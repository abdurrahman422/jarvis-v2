# Jarvis Project File Explanation

Generated on: 2026-05-01

This document explains the Jarvis desktop assistant project file by file. It was written for a student developer, so the language is intentionally direct and practical.

Ignored while scanning: .venv, .git, __pycache__, build, dist, node_modules, *.pyc, runtime/audio/cache, runtime/cache, and temporary log files. Runtime screenshots/audio and model checkpoints are documented as generated artifacts instead of being treated as normal source code.

## Project Overview

Jarvis is a Windows desktop assistant built with Python and PySide6. The UI is in app/ui, the main command brain is in app/core, services live in app/services, persistent data access is in app/data, and datasets/models are in assets, app/generated, and app/models.

The main entry point is app/main.py. It configures Windows/Qt runtime settings, logging, offline mode, database tables, then creates a PySide6 QApplication and shows MainWindow. MainWindow owns the page navigation and connects UI events to AssistantController. AssistantController is the central command router: it receives text or speech, normalizes it, checks hard system actions first, checks alias dataset actions, runs local intent routing, calls automation/system services, optionally asks the AI brain, saves conversations/commands, and sends replies to TTS.

Main architecture layers:
- UI layer: app/ui/main_window.py and app/ui/pages/*.py.
- Controller/routing layer: app/core/assistant_controller.py, intent_router.py, command_normalizer.py, dataset matchers.
- Service layer: app/services/* for speech, system actions, automation, web, AI, hardware, terminal, vision.
- Data layer: app/data/db.py and repository classes.
- Datasets/models: assets/system_action_dataset_500.jsonl, app/generated/bd_full_alias_dataset.json, app/models/*.

## Folder Structure

- app/: Main Python package for Jarvis.
- app/core/: Command routing, intent matching, response formatting, alias/dataset matching, and the central AssistantController.
- app/data/: SQLite setup and repository classes for commands, conversations, settings, alarms, contacts, analytics, and terminal history.
- app/generated/: Generated command alias dataset used by the alias matcher.
- app/models/: Local/offline model artifacts. Qwen LoRA and Bengali speech model files live here.
- app/runtime/: Generated runtime files such as latest TTS audio and screenshots. Mostly safe to regenerate.
- app/services/: Operational services: AI providers, automation tools, speech, system control, web search, vision, hardware, terminal.
- app/ui/: PySide6 UI shell, reusable components, theme constants, and page widgets.
- assets/: Static datasets and voice assets.
- data/: SQLite database and runtime app logs. Database is user data.
- scripts/: Utility and test scripts.
- tests/: Python test package placeholder.

## Execution Flow

When you run:
python -m app.main

1. Python imports app/main.py.
2. main.py configures Qt/Windows process environment and DPI behavior.
3. Logging is configured, normally writing into data/logs.
4. Offline guard is installed if OFFLINE_MODE is enabled.
5. Startup diagnostics check environment/dependencies.
6. The database is initialized through app/data/db.py.
7. A PySide6 QApplication is created.
8. MainWindow from app/ui/main_window.py is created.
9. MainWindow creates AssistantController using app/core/assistant_controller.py.
10. AssistantController creates repositories, STTService, TTSService, AdvancedBrain, ActionRegistry, and loads defaults.
11. MainWindow creates pages from app/ui/pages/*.py and puts them into the navigation stack.
12. User input from HomePage or voice mode is sent to AssistantController.process or listen_once_and_process.

## Voice Flow

1. User clicks microphone or voice loop captures audio.
2. AssistantController.listen_once_and_process calls STTService.transcribe_with_retries.
3. STTService calls GoogleSTTService.
4. GoogleSTTService configures SpeechRecognition settings: ambient adjustment, energy threshold, pause threshold, phrase threshold, and non-speaking duration.
5. Audio is cleaned before Google STT: silence trim, volume normalization, too-short/noisy rejection.
6. Google returns text.
7. AssistantController.process receives recognized text.
8. Text is normalized and corrected for common STT mistakes like "voliom" -> "volume".
9. Hard system actions are checked first.
10. Alias dataset actions are checked second.
11. Local intent routing and AI brain fallback happen after that.
12. The response is saved to command/conversation repositories.
13. TTSService speaks the reply if voice replies are enabled.

## UI Flow

1. app/main.py creates MainWindow.
2. MainWindow imports and creates page classes:
   - HomePage
   - VoicePage
   - VoiceModePage
   - SchedulerPage
   - BrainPage
   - ApiBrainPage
   - AutomationPage
   - SettingsPage
   - VisionPage
3. Shared styles come from app/ui/theme.py.
4. Reusable cards/buttons/status widgets come from app/ui/components.py.
5. UI events call methods on MainWindow.
6. MainWindow calls AssistantController for actual logic.
7. Page widgets display returned dict fields like response, action, target, success, path, and recognized_text.

## System Action Flow

Examples: volume, mute, unmute, screenshot, Bluetooth settings, folders, shutdown confirmation, CPU usage, internet check.

1. AssistantController.process receives text.
2. normalize_voice_command cleans it.
3. _apply_voice_text_correction handles common voice mistakes.
4. _try_volume_clarification catches bare "volume"/"ভলিউম".
5. _try_dataset_system_action calls match_dataset_action.
6. system_action_dataset_loader checks hard rules before dataset/fuzzy matching.
7. A SystemActionExample record is returned with id, intent, action, target, response.
8. execute_dataset_action runs system_action_executor.execute_system_action.
9. system_action_executor logs matched action and executes a safe mapped function.
10. Dangerous power actions create pending confirmation instead of immediate execution.
11. Results go back to UI using executor response, not static dataset response.

Important mappings:
- volume_down, volume_up, mute use safe key fallback when configured.
- volume_25/50/75/100 use pycaw exact endpoint volume.
- screenshot saves to app/runtime/screenshots.
- bluetooth_on/off opens Windows Bluetooth settings.
- Downloads/Desktop open known folders.
- cpu_usage/ram_usage/battery/storage use psutil/shutil.
- internet_status tries a socket connection to google.com:443.

## AI/Brain Flow

1. AssistantController first tries deterministic actions.
2. If no system/action/alias route handles the input, IntentRouter checks local rules.
3. If local intent confidence is low or the message looks conversational, AdvancedBrain may handle it.
4. AdvancedBrain uses app/services/ai provider routing.
5. LocalFallbackProvider can answer without internet.
6. QwenLoraProvider can use app/models/jarvis-qwen-lora if USE_QWEN_LORA / USE_TRAINED_QWEN is enabled and dependencies are installed.
7. Offline guard can block internet providers when OFFLINE_MODE=true.
8. Older Groq/Gemini style provider names are referenced in the project history/request context, but no active groq_client.py file was present in the current scan. Treat Groq/Gemini as not fully confirmed unless reintroduced in providers/settings.

## Config / Environment Variables

From .env:
- OFFLINE_MODE: true/false. When true, internet-dependent services should avoid network usage.
- ENABLE_TTS_SELF_TEST: controls TTS startup self-test.
- BENGALI_TTS_PROVIDER: selects Bengali TTS provider behavior, currently set to offline.
- USE_QWEN_LORA: enables Qwen LoRA provider path.
- USE_TRAINED_QWEN: enables trained Qwen model behavior.
- QWEN_BASE_MODEL: base Hugging Face model name.
- QWEN_LORA_PATH: local LoRA adapter path.
- JARVIS_PERSONA_STYLE: response persona style, currently bn_sir.
- VOSK_MODEL_PATH: local Bengali speech model path.
- PIPER_EXE: optional Piper executable path.
- PIPER_BN_MODEL: optional Piper Bengali model path.

Other settings are stored in data/jarvis.db through SettingsRepository, including STT language, response language, voice reply enabled, wake word enabled, noise reduction enabled, mic sensitivity, Bengali TTS speed, voice id, voice rate, and voice volume.
